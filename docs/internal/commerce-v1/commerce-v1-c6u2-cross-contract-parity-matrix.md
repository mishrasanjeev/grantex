# Commerce V1 C6U2 Cross-Contract Parity Matrix

## Status

This is an internal planning and release-control document. It does not approve production launch, real merchants, public discovery, checkout/payment creation, live payments, live Plural, provider calls, merchant private API calls, production config, production allowlists, cloud resources, protocol publication, external submission, certification, compliance, conformance, standardization, public-launch readiness, merchant approval, or production readiness.

The purpose is to prevent drift between Grantex-owned Commerce V1 contracts and AgenticOrg buyer-agent assumptions. Missing or gated items below must become future implementation slices before any live or public path is considered.

| Risk | Why it matters | Owner | Priority | Next slice |
| --- | --- | --- | --- | --- |

| Public discovery semantics are split between Grantex and AgenticOrg gates | Grantex has well-known/profile and C6G preview gates; AgenticOrg has a separate environment gate. A future launch needs a shared release checklist and compatible state names. | Both | P0 | C6U5 cross-repo discovery state contract. |
| Standards adapter outputs are internal previews only | schema.org/UCP-style/ACP-style/AP2-style previews exist, but no external protocol packaging contract or public claim is approved. | Grantex | P1 | C6U10 adapter packaging readiness without publication. |

## Grantex-Owned Follow-Up Slices

1. C6U3: Cross-repo alias schema tests for MCP/REST aliases, including `payment.get_status` passport expectations and consent field naming.
2. C6U4: Buyer-safe catalog, source, freshness, price/tax/warranty/return projection from Grantex to AgenticOrg without private data.
3. C6U5: Public discovery state contract across Grantex well-known profile, C6G handoff, and AgenticOrg discovery gate.
4. C6U6: Consent/passport/session/revocation propagation contract and channel cache invalidation evidence.
5. C6U7: Order foundation with line-item snapshot, payment reference, support reference, status, and audit.
6. C6U8: Fulfillment, delivery, support, return/refund, and dispute handoff contracts.
7. C6U9: Sandbox checkout E2E after order foundation, using mock or sandbox provider only.
8. C6U10: Internal adapter packaging for schema.org/UCP-style/ACP-style/AP2-style/MCP/A2A without external publication or certification claims.

## Stop Conditions

- Any future change exposes checkout/payment, live provider, live Plural, public discovery, production allowlists, provider credentials, merchant private APIs, or production config before a fresh approval.
- Any future AgenticOrg change calls a provider, Plural, or merchant private API directly instead of Grantex.
- Any future adapter or docs change claims external protocol status, certification, compliance, conformance, standardization, public-launch readiness, merchant approval, or production readiness.
- Any future buyer-facing channel omits refusal handling for missing consent, disabled merchant/agent, policy denial, stale inventory, public discovery disabled, checkout not enabled, live provider not enabled, merchant private API blocked, or provider call blocked.
